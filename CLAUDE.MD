# CLAUDE.md

## Project
This repository is for a polished MVP-style demo website for a defence innovation platform called "Bharat Defence Network". The site is meant to look realistic, investor/demo-ready, and production-quality, but much of the content is dummy/sample data.

## Core Goal
Build a visually strong, minimal, premium web platform that showcases a future national defence innovation ecosystem.

The website should present these ideas as if the platform already exists at scale:
- Defence problem marketplace
- Community collaboration hub
- Project/repository showcase layer
- Creator/video/reels showcase layer
- Startup onboarding and funding layer
- Patent/IP support layer
- Semi-annual summit/events layer
- Member login and private community access
- AI assistant/chat area for members

## Product Positioning
The platform should feel:
- premium
- minimal
- modern
- serious
- strategic
- defence-tech oriented
- startup-friendly
- not noisy
- not gimmicky

Think:
- Palantir-style seriousness
- Linear / Vercel cleanliness
- modern dark UI
- crisp motion
- polished typography
- realistic dummy data at scale

## Brand Direction
Brand name: Bharat Defence Network

The platform should feel like a serious Indian defence innovation ecosystem with global product quality.
It should be proud, strategic, and national in mission, but not loud, cliché, or visually overdone.

Avoid:
- cheesy patriotic styling
- excessive flag usage
- noisy “military” graphics
- camo-heavy design
- jingoistic language

Prefer:
- premium dark UI
- modern strategic language
- clean typography
- subtle Indian identity through content and positioning, not visual excess

## What This Website Is
This is a front-end-heavy MVP demo site meant to communicate vision clearly and impress founders, investors, ecosystem partners, universities, and defence stakeholders.

It is NOT meant to be a fully working production backend.

## What To Build
Build the site as a complete polished experience with realistic fake/sample data.

### Main public sections

2. Problems marketplace
3. Projects / solution repositories

5. Creator showcase / videos / reels
6. Startup, funding, and patent support

## Design Rules
- Use Next.js
- Use TypeScript
- Use Tailwind CSS
- Use App Router
- Use clean reusable components
- Prefer server components where sensible
- Keep code organized and production-like
- Use dummy/sample data stored locally in structured files
- Build polished UI with strong spacing and hierarchy
- Avoid clutter
- Avoid cheap military styling
- Prefer dark graphite / slate / subtle accent palette
- Motion should be restrained and premium

## Visual Direction
The UI should feel like a real, modern defence-tech platform for India with global standards.
Use:
- dark background
- soft borders
- subtle card layering if tasteful
- clear navigation
- crisp icons
- premium dashboard sections
- realistic metrics and stats

## Dummy Content Rules
All data can be fictional/sample content, but must feel realistic.
Include:
- sample problems
- sample defence startups
- sample project repos
- sample contributor profiles
- sample event/summit cards
- sample videos/thumbnails
- sample discussions/community stats
- sample AI chat prompts

Do not present fictional companies or claims as real-world verified facts.
Keep clearly demo-safe while still looking realistic.

## Security / Policy Direction
Do NOT include:
- real classified information
- sensitive military instructions
- explicit weapon construction instructions
- harmful operational details
- anything that makes the site look like it facilitates illegal weapons development

Keep the platform framed around:
- innovation
- defence technology ecosystem
- research
- simulation
- collaboration
- talent
- funding
- national capability building
